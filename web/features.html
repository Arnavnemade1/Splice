Capabilities

Everything an agent needs
to survive real web apps.

One MCP server. Nine capabilities that turn brittle click-and-hope automation into something you can actually depend on.

Verified actions

Say what you want in plain language. Splice picks the target, checks preconditions, acts — then proves the outcome, or tells you it didn't happen.

compile_verified_action {
  intent: 'click "Place order"',
  expect: [{ url_contains: "/thanks" }]
}
→ verified ✓  reached /thanks

Failure diagnosis

Stuck agents ask "what state is this page in?" and get an answer — blocked by a modal, failing validation, waiting on the network — with the next move.

Token-efficient observation

After the first look, agents receive only what changed. Framework re-renders are filtered out instead of re-read as new content.

Recovery memory

When a fix works — closing that cookie banner, dismissing that upsell — Splice remembers it per site and suggests it the next time.

Security guardrails

Prompt-injection text is redacted before your agent reads it, secrets are blocked from leaving in requests, and session state is encrypted at rest.

Self-healing runtime

Browser crashes recover automatically with state restored. A journal records every call, so any run can be debugged after the fact.

And more

Three quieter tools that compound over time.

Behavior reportsReplay a session as a readable chain of thought with scores and fixes.
Prompt optimizationVerbose instructions rewritten into terse, grounded intents — only with permission.
Accessibility auditsScored WCAG checks with concrete fixes. The pages agents struggle with are the same ones that fail them.

All nine, in one server.

No config sprawl. Clone, build, connect — the doctor confirms it's healthy.
