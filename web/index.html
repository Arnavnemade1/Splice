Why Splice

Most browser tools stop at execution. Splice is cognition.

Execution-focused tools

  • ✓ Click, type, navigate
  • ✓ Raw DOM or screenshots
  • ✗ Why did that fail?
  • ✗ Did it actually work?
  • ✗ What should I try next?

Splice

  • ✓ Click, type, navigate
  • ✓ Semantic understanding
  • ✓ Diagnose failures with evidence
  • ✓ Verify outcomes
  • ✓ Recommend recovery patterns
  • ✓ See what changed (deltas)
  • ✓ Real-time performance steering

The problem agents face

Stale refs. Hidden modals. Disabled buttons. Route transitions. Login expiry. Silent clicks. Form validation traps. CAPTCHAs.

Most tools expose more page data. Splice exposes browser understanding.

Core Capabilities

Browser intelligence built for real-world complexity

◆

Diagnose

Understand why actions fail. Splice produces forensic evidence: what changed, why it matters, and what to try next.

▶

Verify

After every action, confirm it worked. Compile intent into preconditions, execution plan, and post-action assertions.

↻

Recover

Remember what worked. Recovery memory learns domain-specific patterns and surfaces them when you need them.

Δ

Observe Efficiently

First read gets the full tree. Subsequent reads receive only what changed—added elements, text mutations, URL transitions.

⚡

Optimize Runtime

Token efficiency engine detects wasted reads mid-run and steers your agent toward cheaper observations.

🔒

Defend

Redact prompt injection attempts and hostile page content before page text reaches your agent.

Designed for developers

Fast setup. Clear visibility. Intentional control.

1

Initialize

splice init scaffolds your config and prints ready-to-paste MCP snippets for Claude Desktop, Claude Code, or Cursor.

2

Verify

splice doctor health-checks Node, Chromium, workspace state, and configuration before your first run.

3

Launch

splice start runs the MCP server. Your agent immediately gains access to browser capabilities and real-time diagnostics.

4

Observe

Open the Command Center dashboard for live per-agent metrics, action audit trails, token usage, and exportable evidence.

Example: Compile and verify an action
{ "tool": "compile_verified_action", "input": { "intent": "Log in with credentials", "constraints": ["avoid_otp", "production_account"], "execute": true } } // Response includes: // - preconditions (login form visible?) // - plan (click email → type → click submit) // - postconditions (redirect to dashboard?) // - evidence (screenshots, state changes) // - confidence (80%)

Works with your stack

Splice is an MCP server. Integrate it anywhere the MCP SDK runs.

Claude Desktop

Add Splice to your claude_desktop_config.json and use it in conversations with Claude.

"args": ["node", "/path/to/Splice/dist/index.js"]

Claude Code

Equip your coding assistant with browser capabilities and real-time diagnostics.

splice start # then add to MCP servers

Custom Agents

Use the MCP SDK to connect your own agents. Splice exposes all tools over stdio.

StdioClientTransport({ command: "node", args: [...] })

OpenClaw Gateway

For external systems, Splice can expose a WebSocket gateway with full capability access.

OPENCLAW_GATEWAY_PORT=18789

Built for scale

Enterprise-ready observability and reliability

Delta Observations

Incremental snapshots return only changes. Full-tree fallback when baselines drift. Save tokens on long sessions.

Recovery Memory

Patterns decay over time (half-life: 14 days). Proven fixes surface when needed. Anonymized telemetry optional.

Token Efficiency

Per-agent token accounting with real-time optimization directives. Detect wasted reads and steer toward cheaper paths.

Runtime Reliability

Crash self-healing. Reproducible run journals. Live per-agent tracking with corrective directives mid-run.

Command Center

Exportable audit evidence. Real-time state changes. Tokens saved. Per-agent analytics dashboard.

Security First

Redaction engine. Secret egress firewall. Hostile content filtering. Deterministic, auditable on-page logic.

Configuration & CLI

Three-layer config resolution with source attribution

Environment variables

Settings resolve in order: environment variables > splice.config.json > defaults. Every configuration source is visible with splice config --json.

Setting
Environment Variable
Description
autoOpenDashboard
SPLICE_AUTO_OPEN_DASHBOARD
Open Command Center on startup
enableOpenClaw
SPLICE_ENABLE_OPENCLAW
Enable WebSocket gateway
watchMode
SPLICE_WATCH_MODE
Launch visible Chromium window

CLI Commands

splice init [--openclaw] [--dashboard] [--watch] Scaffold config and print client wiring
splice doctor Health check: Node, Chromium, ports, workspace
splice start [--openclaw] Run the MCP server
splice config [--json] Show effective configuration with sources

Ready to go

Get Splice running in three commands

$ npm install splice
$ splice init
$ splice start
