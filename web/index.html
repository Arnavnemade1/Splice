How it works

Three steps, then your agent stops guessing.

  1. 1

    Install

    Clone, build, and health-check. The doctor tells you exactly what to fix if anything is off.

    git clone https://github.com/Arnavnemade1/Splice.git
cd Splice && npm install && npm run build
node dist/cli.js doctor
  2. 2

    Connect your agent

    One line for Claude Code. splice init prints snippets for Claude Desktop and Cursor.

    claude mcp add splice -- node dist/cli.js start
  3. 3

    Let it work

    The agent diagnoses each page before acting, executes intents that are verified after the fact, and reads only what changed — not the whole page, every time.

    compile_verified_action { intent: 'click "Place order"',
  expect: [{ kind: "url_contains", value: "/thanks" }] }

What you get

Everything an agent needs to survive real web apps, in one server.

Verified actions

Say what you want in plain language. Splice picks the target, checks preconditions, acts, then proves the outcome — or tells you it didn't happen.

Failure diagnosis

Stuck agents ask "what state is this page in?" and get an answer: blocked by a modal, failing validation, waiting on the network — with the recommended next move.

Token-efficient observation

After the first look at a page, agents receive only what changed. Framework re-renders are filtered out instead of read as new content.

Recovery memory

When a fix works — closing that cookie banner, dismissing that upsell — Splice remembers it per site and suggests it the next time the same wall appears.

Behavior reports

Every session can be replayed as a readable chain of thought with scores and specific recommendations, so the next run is better than the last.

Prompt optimization

Verbose instructions get rewritten into the terse, grounded intents that rank best — only with permission, never silently.

Accessibility audits

Scored WCAG checks with concrete fixes. Pages that fail them are exactly the pages agents struggle with — fixing one fixes both.

Security guardrails

Prompt-injection text is redacted before your agent reads it, secrets are blocked from leaving in requests, and session state is encrypted at rest.

Self-healing runtime

Browser crashes recover automatically with state restored. A journal records every call so any run can be debugged after the fact.

Agents install it themselves

Point any coding agent at the install guide. Every step is a command with a machine-checkable success criterion — the agent isn't done until splice doctor --json says "healthy": true. 

Read https://github.com/Arnavnemade1/Splice/blob/main/AGENT_INSTALL.md
and follow it. Verify with `npx splice doctor --json` before reporting success.

That's the whole prompt. Once connected, the agent reads the built-in playbook resource and knows the loop: diagnose → act verified → confirm → recover.
